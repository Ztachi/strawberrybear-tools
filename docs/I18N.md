# 国际化规范

## 规则

所有项目必须做国际化，初始只适配中英文（zh-CN / en-US）。

## 实施要求

1. 所有用户可见的文本必须使用国际化 key
2. 不允许硬编码任何用户可见的文本
3. 默认语言为中文（zh-CN），英文（en-US）作为可选适配
4. 翻译文件统一存放在项目根目录的 `locales/` 或 `i18n/` 目录下
5. 使用成熟的 i18n 库（如 vue-i18n、react-i18next）

## 目录结构示例

```
src/
  ├── i18n/
  │   ├── index.ts
  │   ├── zh-CN.ts
  │   └── en-US.ts
  └── locales/
      ├── zh-CN.json
      └── en-US.json
```

## 翻译 Key 命名规范

使用点号分隔的层级结构：
- `common.button.confirm` - 通用按钮-确认
- `common.button.cancel` - 通用按钮-取消
- `page.home.title` - 页面-首页-标题
