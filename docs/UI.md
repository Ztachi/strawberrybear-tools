# UI 交互规范

## 文本超长处理

### 规则说明

当文本可能超出容器宽度或显示行数时，必须同时满足以下条件：

1. **视觉限制**：使用平台/框架的截断类工具限制显示（如 Tailwind CSS 的 `truncate` 单行截断或 `line-clamp-*` 多行截断）
2. **悬停提示**：外层包裹工具提示组件，鼠标悬停时显示完整文本

### 适用场景

| 场景          | 处理方式           | 说明                               |
| ------------- | ------------------ | ---------------------------------- |
| 列表项名称    | 单行截断 + Tooltip | 文件名、歌曲名等可能超长的单行文本 |
| 详情页标题    | 多行截断 + Tooltip | 允许显示2-3行，超出部分截断        |
| 用户输入内容  | 截断 + Tooltip     | 任何由用户输入、可能较长的文本     |
| 卡片/面板标题 | 根据容器宽度自适应 | 通常单行截断即可                   |

### 设计原则

- **避免截断丢失信息**：所有被截断的文本都必须能通过悬停/点击等方式查看完整内容
- **一致性**：同类文本采用相同的截断策略
- **上下文适配**：根据内容类型和显示容器选择合适的截断方式

### 注意事项

- 不同技术栈的截断实现方式不同（Vue/React/小程序等），按各框架最佳实践实现
- Tooltip 仅用于补充展示，不应作为主要信息展示方式
- 移动端需考虑触摸长按的替代方案
