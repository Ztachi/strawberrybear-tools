# 架构概述

## 系统架构

Infinity Nikki Player 采用 Tauri 2.x 架构，分为前端（Vue 3 + TypeScript）和后端（Rust）。

```
┌─────────────────────────────────────────────────────────┐
│                    Frontend (Vue 3)                      │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌──────────┐  │
│  │  Views  │  │Components│  │ Stores  │  │   i18n   │  │
│  └────┬────┘  └────┬────┘  └────┬────┘  └──────────┘  │
│       └────────────┴────────────┘                        │
│                        │ invoke()                         │
├────────────────────────┼────────────────────────────────┤
│                    Tauri IPC                              │
├────────────────────────┼────────────────────────────────┤
│                        │                                  │
│  ┌─────────────────────┴───────────────────────────────┐  │
│  │               Backend (Rust)                        │  │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────────────┐ │  │
│  │  │ Commands │  │   MIDI   │  │ Keyboard Sim      │ │  │
│  │  │          │  │  Parser  │  │ (enigo)           │ │  │
│  │  └──────────┘  └──────────┘  └──────────────────┘ │  │
│  └────────────────────────────────────────────────────┘  │
│                        │                                  │
│  ┌─────────────────────┴───────────────────────────────┐  │
│  │           OS Layer (macOS / Windows)                 │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

## 模块职责

### 前端 (src/)

| 目录          | 职责                           |
| ------------- | ------------------------------ |
| `views/`      | 页面级组件（主窗口、悬浮窗）   |
| `components/` | 可复用组件（播放器、控制条等） |
| `stores/`     | Pinia 状态管理                 |
| `i18n/`       | 国际化配置                     |
| `lib/`        | 工具库（MIDI 播放）            |

### 后端 (src-tauri/src/)

| 文件/目录   | 职责                         |
| ----------- | ---------------------------- |
| `lib.rs`    | 应用配置、菜单、系统语言检测 |
| `main.rs`   | 程序入口                     |
| `commands/` | Tauri 命令实现               |
| `midi/`     | MIDI 解析和处理              |
| `keyboard/` | 键盘事件模拟                 |
| `types.rs`  | 共享类型定义                 |

## 窗口模型

| 窗口     | Label     | 用途         |
| -------- | --------- | ------------ |
| 主窗口   | `main`    | 完整功能界面 |
| 悬浮窗口 | `overlay` | 极简播放控制 |

窗口通过 URL 参数 `?windowLabel=xxx` 区分。

## 状态管理

使用 Pinia 管理前端状态：

- `usePlayerStore` - 播放状态、当前 MIDI 文件
- `useMidiStore` - MIDI 库、文件列表

后端使用 `parking_lot::Mutex` 管理共享状态：

- `AppState` - 全局应用状态
- `PlayerControl` - 播放控制状态
